## Read about it
#### What? 
- This is a simple copy/paste button <img width="199" height="32" alt="image" src="https://github.com/user-attachments/assets/9512a9eb-1dd8-4835-971d-5fd3b6d13f61" />
 (built with Claude.ai) that takes clipboard content and inserts into a template for pasting elsewhere. It increases the efficiency of this microtask by over 45% on average
#### Why? 
- Copying and pasting has its place. But why do it more often than you need? If you can be more efficient (and accurate) then automate it, and [get 1% better](https://jamesclear.com/marginal-gains) (seriously, check out the article at that link).
#### How?
- Automate. As simple as this automation is, it is exponentially more powerful when combined with similar improvements on workflows. This is an *investment* of time whose returns compound hourly.
<br>

### ⏲️ Original steps (manual): 🐢 9 - 14 seconds

1. Copy the text (triple-click is most efficient) and Ctrl + C to copy (1 second)
2. Navigate to note field (2 seconds)
3. Type the phrase and punctuation (3 seconds)
4. Click to place the cursor, or arrow key to the middle of quotation marks (1-3 seconds)
5. Ctrl + V to paste (1 second)
6. Correct formatting if the copied text adds a return or extra spaces (0 - 3 seconds)
7. Save the note (1 second)

### 🔥 After button  implementation: 🐇 6 seconds

1. Copy the text (triple-click is most efficient) and Ctrl + C to copy (1 second)
2. Click the button (automatically inserts clipboard content into the template, formats properly, places the complete note on clipboard) (1 second)
3. Navigate to note field (2 seconds)
4. Ctrl + V to paste (1 second)
5. Save the note (1 second)

##### ⚠️ Risks

- Original method risks typos and inconsistent formatting
- Original method takes more time (❗3 - 8 additional seconds per note, per user❗)

##### 📈 Advantages

- The button eliminates the need to navigate the cursor into an exact position
- The button formats the text, eliminating extra spaces and returns, and places a second line for the closing remarks, per formatting preferences

**_DISCLAIMER:_**
_This assumes the button is open and readily available without navigating to different screens, perhaps implemented as a broswer extension._
